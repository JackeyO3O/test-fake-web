假 API 文件範例

1. 取得用戶資訊 API

說明
此 API 用來取得指定用戶的詳細資訊。

請求

GET /api/v1/users/{userId}

路徑參數

參數名稱 類型 必填 說明
userId string 是 用戶唯一識別碼

成功回應

{
  "userId": "12345",
  "name": "John Doe",
  "email": "john@example.com",
  "createdAt": "2025-07-14T12:34:56Z"
}

錯誤回應

{
  "error": "User not found"
}

2. 建立訂單 API

說明
此 API 用來為用戶建立一筆新的訂單。

請求

POST /api/v1/orders
Content-Type: application/json

請求體（JSON）

{
  "userId": "12345",
  "items": [
    { "productId": "abc123", "quantity": 2 },
    { "productId": "def456", "quantity": 1 }
  ],
  "note": "請盡快出貨"
}

成功回應

{
  "orderId": "order7890",
  "status": "pending",
  "createdAt": "2025-07-14T13:00:00Z"
}

錯誤回應

{
  "error": "Invalid product ID"
}
